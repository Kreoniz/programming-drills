# Webpack Aliases and Source Maps

## Learning Objectives
- Configure resolve aliases
- Support TS extension resolution
- Choose production source maps intentionally

## Task
Update the webpack config with `@app` and `@shared` aliases, TS/TSX extensions, and hidden source maps for production diagnostics.

Edit files only under `starter/` while practicing. The `solution/` directory is a reference implementation used by the verifier.

## Expected Commands
- From this exercise: `node ../tests/check.mjs .`
- From the repository root: `node tools/drill-runner.mjs check ts-react/webpack/003-alias-source-maps`
- Reference check: `node tools/drill-runner.mjs check ts-react/webpack/003-alias-source-maps --solution`
